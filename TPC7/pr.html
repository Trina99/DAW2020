Abstract

Neste TPC é suposto completar o mais possível a aplicação já inicializada na última aula prática.

Como tal, foram criadas as seguintes rotas:

  • GET /students - rota já feita inicializada na aula, que apresenta a lista de todos os estudantes;
  • GET /students/:id - devolve a página indivial do aluno que mostra se os 8 TPCs foram ou não realizados;
  • GET /students/register - responde com um formulário para adicionar um novo estudante;
  • GET /students/edit/:id - responde com o formulário pré-preenchido e permite que o utilizador altere os valores;
  • POST /student - recebe os dados do formulário (req.body), e insere um registo na BD;
  • POST /student/edit/:id - recebe os dados dum formulário e altera o registo do aluno na BD;
  • POST /student/delete/:id - apaga o aluno da BD;

Para a resolução deste trabalho foi utilizada uma base de dados em Mongo, um servidor em Nodejs com extensão express e para view, as páginas foram originadas na linguagem Pug.
